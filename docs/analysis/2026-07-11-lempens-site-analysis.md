## 体験の構造（観察結果）

1. **エントリーゲート**: プリロード画面（ランドマークのシルエット + STARTボタン +
   「Turn your sound on / Switch to desktop」）。STARTクリックで初めて体験開始
   （= AudioContext のユーザージェスチャー要件を満たす設計）。
2. **1本道の旅**: 主人公アバター（本人の3D分身）がパリの街に立ち、スクロールすると
   カメラが川沿い→橋→広場と**街の中を前進**する。移動区間ではアバターがスクーターに乗り、
   カメラがその後ろを追走する（=「旅」のメタファーを乗り物で具現化）。
3. **国=セクション**: France → Luxembourg → Belgium の3章構成（コンソールログで
   コンポーネント名を確認）。国の切り替えは
   **白ワープ（放射状の光線）→ 液体インクマスクで次の街がにじみ出るリビール**。
   ページ遷移感ゼロで「長距離移動」を表現しつつ、シーンの load/unload を隠す実用も兼ねる。
4. **セクションタイトルは3D押し出し文字**（例: "ABOUT ME" が広場に建造物のように鎮座）。
   世界の中に文字が「存在」しており、UIオーバーレイではない。
5. **テキストコンテンツは半透明ダークパネルのHTMLオーバーレイ**（DOM構造に `__html`
   プレフィックスのクラスが多数 = R3F/drei系実装の痕跡）。スクロールに応じた staggered リビール。
6. **「SCROLL」プロンプト**が各章の頭で表示され、ユーザーを誘導する。

## クオリティの源泉（★2質感に対する学び）

- 質感の正体は**「複雑なマテリアル」ではなく「統一されたカラーグレーディング」**。
  ローポリ造形 + シーン全体をピンク/コーラルの夕景1トーンで染め、木々の緑だけを
  アクセントに残す。フォグ・ソフトシャドウ・夕日のリムライトで空気感を出している。
- **教訓: 個々のオブジェクトの material を盛るより、シーン単位の「色の支配」
  （背景色・フォグ・ライト・トーンマッピング）を先に決めるほうが質感への寄与が大きい。**

## 追記: Phase 2 質感テストの結果（2026-07-11）

1. **クリスタル球の見た目は既存 `Crystal.tsx` のレシピが正解**とユーザー確認済み。
   新規にマテリアルを発明した2案（粗い20面ジェム / 不透明ツヤ球）はいずれも却下。
   「ゼロベース実装」はアーキテクチャの話であり、ボールの見た目=ブランドは既存を移植する。
   レシピ: icosahedron[1.5,2] + meshPhysicalMaterial(transmission 0.7, flatShading) +
   オレンジコア(パルス3.5±1.2) + 白熱センター + Bloom(1.8/0.3/0.9)。
2. **背景・シーンデザインの刷新はユーザー承認済み**。「パリのサイトのような雰囲気でもいい。
   球体の雰囲気が合わなければ少し変えてもいい」「暗くなくていいよ」(2026-07-11)。
   Phase 2では**Lempens風の明るいパステル夕景**に振り切り、以下の構成で成立を確認:
   - drei `Sky`(太陽を地平線近くに配置した物理夕焼け空) + `Clouds`(ピーチ色の雲塊)
   - フォグ #f2b8a0(空の地平線色と一致) / 地面 #c8a9a3(乾いたグレージュ・低彩度)
   - 夕日キーライト #ffb185 + 薄紫リムライト #c3b0ff(球の輪郭を背景から立てる)
   - Bloom intensity 1.1 / threshold 0.9(空の暴発なしで太陽と白熱コアだけ滲む)
   - design-reviewスキルの指摘3件(地面の彩度・リムライト・Bloom)を反映済み
   Phase 3のセクション実装時に、この方向でアートディレクションを本格展開する。
   Phase 3送りの指摘: 青い箱/青オーブの色設計・ラベル可読性・375px検証・
   prefers-reduced-motion・マイクロインタラクション。
