# 要件定義

## プロジェクト概要

**名称:** yusu portfolio  
**目的:** 体育教師→エンジニアへのキャリアチェンジストーリーを持つポートフォリオ。作ったすべての個人プロジェクトのハブ。  
**対象ユーザー:** 採用担当者・副業依頼者・エンジニアコミュニティ  
**GitHub:** https://github.com/yusu31/portfolio  
**Issue:** https://github.com/yusu31/portfolio/issues/1

---

## 機能要件

### ページ構成

| セクション | 内容 | 優先度 |
|---|---|---|
| Hero | キャッチコピー + 「元体育教師のエンジニア」を冒頭に出す + CTAボタン | 必須 |
| About | 転職ストーリー（時系列ではなく「何を学んだか」で語る）+ 体育→エンジニアスキル対比 | 必須 |
| Works | プロジェクトカード（デモURL + GitHub + ケーススタディ） | 必須 |
| Skills | 技術スタックアイコン + 習熟度 | 必須 |
| Blog | ZennへのOGP付きリンク一覧 | 中 |
| Contact | メールフォーム（Resend API） | 必須 |

### Works セクション詳細

各プロジェクトカードに以下を記載:
- なぜ作ったか（課題・動機）
- どう解決したか（技術選定の理由）
- 何を学んだか
- 動くデモURL（必須）+ GitHub URL（必須）

最低3本、うち1本はスポーツ・教育ドメイン（TAIIKU BASE等）

### About セクション詳細

教師時代スキルをエンジニア文脈で語る:

| 教師時代のスキル | エンジニアとしての言い換え |
|---|---|
| 授業設計（目標→手段→評価の構造化） | 要件定義・設計能力 |
| 生徒にわかりやすく説明 | ドキュメント作成・チーム内教育力 |
| 体力・忍耐力・PDCA改善 | 粘り強いデバッグ・改善サイクルへの適性 |
| チームスポーツの指導 | チームコラボレーション |
| 生徒のモチベーション管理 | UX思考・ユーザー視点 |

### アニメーション要件

| 箇所 | 手法 | 理由 |
|---|---|---|
| Hero（メインビジュアル） | GSAP + ScrollTrigger | スクロールストーリーテリングに最適 |
| フェードイン（各セクション） | CSS Scroll-Driven Animations | 0KB・ブラウザネイティブ |
| ページ遷移 | View Transitions API | 0KB・CSSで映画的遷移 |
| Reactコンポーネント | Framer Motion | React Island内のマイクロインタラクション |

### レスポンシブ要件

- モバイルファースト設計
- ブレークポイント: `sm:640px / md:768px / lg:1024px / xl:1280px`
- スマートフォンでのCore Web Vitalsが採用担当者の第一印象を決める

### 言語対応

- 日本語・英語 併記（グローバル志向を示す）
- デフォルト日本語

### その他機能

- ダークモード / ライトモード切替（ダークファーストで設計）
- OGP設定（SNSシェア時にカード表示）
- sitemap.xml + robots.txt（SEO）
