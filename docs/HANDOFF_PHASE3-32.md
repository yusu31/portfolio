### 4. 社内コード調査(Explore 3体並列)

Plan Mode Phase 1として、以下を並列調査:

### 5. 外部事例調査+実装設計(Fable 5、Plan agent)

three.js/WebGL界隈の実装パターン・ゲームの重力反転演出を調査した上で設計を作成。主な結論:

- **`computeFrenetFrames`(Frenet-Serretフレーム)は不採用**: 経路形状から向きを導出する標準手法だが、直線区間で法線が不安定・反転する既知バグがthree.js本体でも報告されており、「経路を変えずに向きだけ演出したい」という要件と構造的にミスマッチ
- 業界事例(DEPT社の記事等)は軒並み「位置用スプライン+視線ターゲット用スプラインの2本立て」構成を採用しており、本プロジェクトの現行`CAMERA_PATH`+`LOOKAT_PATH`構成と一致。**この骨格は温存し、`lookAt`の後にroll/pitchの「姿勢レイヤー」を重ねる**方式(`camera.rotateX`→`camera.rotateZ`)を採用
- Sky/Fogは物理的にロール・ピッチで破綻しない(Fogは深度ベース・方向非依存、Skyはワールド座標固定)。唯一のリスクは太陽が視錐台に入りBloom(threshold 0.9)が暴発すること

### 6. 設計レビューと4つの確認ポイント

Fable 5案をレビューする中で、最初に見せたビジュアルコンパニオンの図解と食い違う点を発見し、ユーザーに再確認:

4. **実施順序**: コート拡大(Phase 5-5)を先に確定させてから、本機能の数値調整に入る
